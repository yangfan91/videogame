# 电玩计时

一个面向电玩店、游戏包厢和主机娱乐空间的本地桌面计时管理系统。项目基于 `Python + PyQt6 + SQLite` 开发，主要用于前台值班人员管理包厢/设备状态、开启计时、暂停恢复、套餐倒计时、到时提醒、结算登记和查看基础运营统计。

## 项目简介

`电玩计时` 的核心目标是把门店日常的“开台、计时、提醒、结算、记录”流程集中到一个桌面程序里完成。系统不依赖外部服务器，数据保存在本机 SQLite 数据库中，适合单机前台电脑使用。

当前版本更偏向“计时与运营登记工具”，不是完整的自动收费系统。它会记录每次会话的使用时长、付款状态、收款方式和备注，但不会自动根据价格规则计算应收金额。

## 主要功能

- 设备/包厢管理：维护 PS4、PS5、Xbox、Switch 等包厢类型和具体设备。
- 设备控制台：以卡片形式展示全部设备当前状态，支持快速开始、暂停、继续、加时和结算。
- 两种计时模式：支持自由计时和团购/套餐倒计时。
- 到时提醒：倒计时即将结束或已到时时，界面状态会变化，并支持 Windows 语音提醒。
- 会话恢复：程序重启后可恢复未完成的计时会话，减少意外关闭造成的数据丢失。
- 拖拽排序：设备卡片支持拖拽调整显示顺序，并持久化保存。
- 结算登记：记录是否已付款、收款方式、备注和最终使用时长。
- 统计报表：按日期查看历史会话、设备使用情况、模式占比和付款状态。
- 备注维护：可在历史记录中补充或修改备注，方便门店交接和复盘。
- 本地数据存储：使用 SQLite 保存设备、会话、暂停记录和排序信息。

## 技术栈

- Python 3
- PyQt6

## 目录结构

```text
.

├── plans/               # 开发计划文档
├── stitch_exports/      # UI 设计稿/页面原型导出
├── tests/               # 自动化测试
├── ui/                  # PyQt6 界面组件和主窗口
├── config.py            # 全局配置、主题色、路径和状态常量
├── main.py              # 程序入口
├── requirements.txt     # Python 依赖
└── 电玩计时.spec         # PyInstaller 打包配置
```

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行程序

```bash
python main.py
```

程序启动时会自动初始化本地数据库，并在项目根目录下创建或使用：

```text
data/videogame.db
```

### 3. 打包为 Windows 可执行文件

```bash
pyinstaller 电玩计时.spec
```

打包产物会输出到 `dist/` 目录。运行打包后的程序时，数据库会保存在可执行文件同级目录下的 `data/videogame.db`。

## 使用场景

适合以下场景：

- 电玩店前台包厢计时
- 主机游戏体验店设备管理
- 小型娱乐空间的单机计时登记
- 团购套餐时长核销和提醒
- 日常值班交接记录

典型流程：

1. 在设备控制台选择空闲设备。
2. 选择自由计时或套餐倒计时。
3. 顾客使用过程中可暂停、继续或加时。
4. 到时后系统提醒前台处理。
5. 顾客离店时完成结算登记。
6. 店长可在统计报表中查看历史记录和使用情况。

## 当前限制

当前版本暂未实现以下能力：

- 自动金额计算
- 套餐价格管理
- 会员、优惠券和折扣规则
- 多门店管理
- 多电脑实时数据同步
- 权限账号体系
- Excel / PDF 报表导出
- 完整的设备维护状态后台流程

这些功能可以作为后续版本继续扩展。

## 数据说明

系统数据默认保存在本地 SQLite 数据库中：

```text
data/videogame.db
```

建议在升级程序、迁移电脑或批量调整设备前备份该文件。数据库文件属于本地运行数据，默认不会提交到 Git 仓库。

## 开发说明

运行测试：

```bash
pytest
```

项目已配置 `.gitignore`，会忽略打包产物、缓存文件、临时文件和本地数据库，仓库主要保存源码、资源、测试和文档。

## 项目状态

当前项目已经具备门店前台日常使用所需的基础能力，包括设备管理、计时控制、到时提醒、结算登记、统计查询和本地数据持久化。后续可以围绕自动计费、数据导出、权限管理和多端同步继续完善。
