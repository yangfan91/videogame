# 电玩店包厢计时系统 功能手册

## 1. 文档说明

本手册用于说明 `电玩店包厢计时系统` 当前版本的实际功能、页面结构、日常操作流程与注意事项，适用于门店前台、值班人员、店长或系统维护人员快速上手。

本系统是一个基于 `PyQt6 + SQLite` 的本地桌面程序，主要用于：

- 管理店内各个游戏包厢/设备
- 开始、暂停、恢复、加时和结束包厢计时
- 支持“团购套餐倒计时”和“自由计时”两种模式
- 在套餐即将到时或已经到时后提醒
- 记录结算状态、收款方式和备注
- 按日期查看历史使用记录与统计报表
- 在主窗口内维护包厢类型和包厢列表
- 通过拖拽调整包厢卡片显示顺序

## 2. 系统定位

当前程序版本的核心定位是“包厢计时与运营登记系统”，不是自动金额计费系统。

本版本已经实现：

- 包厢/设备基础资料维护
- 包厢卡片状态展示
- 自由计时
- 团购套餐倒计时
- 暂停与恢复
- 套餐加时
- 倒计时预警和到时提醒
- 结算确认
- 付款状态记录
- 收款方式记录
- 备注记录
- 历史记录查询
- 基础统计报表
- 设备卡片拖拽排序
- 程序重启后恢复未完成会话

本版本没有实现：

- 自动按单价计算费用
- 自动生成应收金额
- 自动找零
- 套餐价格管理
- 多级价格规则配置
- 会员、优惠券或折扣体系

因此，结算时系统记录的是“会话时长、付款状态、收款方式、备注”，不直接计算金额。

## 3. 系统启动与数据存储

### 3.1 启动方式

程序启动入口为：

- 开发环境：`main.py`
- 打包后可执行文件：`dist/电玩计时.exe.1`

启动时程序会自动执行以下动作：

- 初始化数据库
- 自动迁移旧版本数据库结构
- 恢复未结束的计时会话
- 打开主窗口

### 3.2 数据库存储位置

系统数据保存在本地 SQLite 数据库中：

- 开发环境：`data/videogame.db`
- 打包后运行：`exe` 同级目录下的 `data/videogame.db`

数据库保存的核心数据包括：

- 包厢类型
- 包厢设备
- 包厢显示顺序
- 计时会话
- 暂停记录

### 3.3 本地运行特性

当前版本是本地桌面程序：

- 不需要外部服务器
- 不需要联网
- 数据保存在本机
- 多台电脑不会自动同步数据

如果门店需要多台电脑共享数据，需要后续增加服务器或局域网数据库方案。

## 4. 主界面说明

主界面由左侧导航栏和右侧内容区组成。

### 4.1 左侧导航

左侧共有 3 个主要入口：

- `控`：设备控制台
- `统`：统计报表
- `设`：系统设置

点击导航按钮后，右侧内容区会在对应页面之间切换。

### 4.2 页面结构

程序主要包含以下 3 个主页面：

1. 设备控制台
2. 统计报表
3. 系统设置

注意：当前版本的系统设置已经嵌入主窗口，不再以单独弹窗展示。

### 4.3 弹窗使用范围

当前仍以弹窗展示的操作包括：

- 开始计时模式选择
- 套餐加时
- 结算确认
- 删除确认
- 错误或提示消息

系统设置本身不再弹窗。

## 5. 设备控制台

设备控制台是系统最常用的页面，用于查看所有包厢状态并执行计时操作。

### 5.1 页面功能

设备控制台支持：

- 展示全部包厢卡片
- 按状态查看设备数量概览
- 直接对包厢进行开始、暂停、继续、加时、结算
- 自动恢复上次未完成的计时会话
- 查看值班摘要、待处理事件和状态图例
- 拖动包厢卡片调整显示位置
- 根据窗口宽度自动调整每行卡片数量

### 5.2 顶部操作

控制台右上角提供：

- `刷新状态`：重新加载设备和会话状态
- `开始新计时`：自动寻找第一个空闲包厢并打开开始计时窗口

### 5.3 状态概览

页面顶部会显示 4 类状态统计：

- 空闲设备
- 进行中
- 已暂停
- 到时提醒

右侧摘要区会显示：

- 当前运行中的设备数量
- 待处理提醒数量
- 全部接入设备数量

### 5.4 包厢卡片信息

每张包厢卡片通常显示以下内容：

- 包厢名称
- 包厢类型
- 当前状态
- 当前计时模式
- 当前显示时间
- 状态说明
- 操作按钮

卡片会根据状态显示不同的状态标签和强调色，方便值班人员快速识别。

### 5.5 支持的设备状态

系统中的包厢状态包括：

- `空闲`
- `使用中`
- `已暂停`
- `已到时`
- `维护中`

说明如下：

- `空闲`：当前没有正在进行的会话，可直接开始计时。
- `使用中`：正在正常计时。
- `已暂停`：本次会话已暂停，计时暂时停止。
- `已到时`：仅用于倒计时模式，表示套餐时间已耗尽，但会话未结束。
- `维护中`：当前版本保留状态支持，但界面中没有完整维护管理入口。

## 6. 包厢卡片排序

### 6.1 自动排布

当前版本不再固定每行只显示两张卡片。

设备列表会根据窗口宽度自动计算可显示列数：

- 窗口较窄时自动减少列数
- 窗口较宽时自动增加列数
- 卡片宽度保持稳定，避免文字被挤压

### 6.2 拖拽调整位置

在设备控制台中，可以用鼠标拖动包厢卡片调整显示顺序。

操作方式：

1. 按住需要移动的包厢卡片。
2. 拖动到目标卡片上。
3. 松开鼠标。

排序规则：

- 拖到某张目标卡片上：移动到目标卡片前面。
- 拖到设备列表空白区域：移动到列表末尾。
- 拖动自己到自己位置：顺序不变。

### 6.3 顺序保存

拖拽调整后，系统会立即将顺序保存到数据库。

保存后：

- 点击“刷新状态”不会丢失顺序。
- 关闭并重新打开程序后仍保持该顺序。
- 新增包厢默认排在当前列表末尾。

### 6.4 注意事项

拖拽只改变显示顺序，不改变：

- 包厢名称
- 包厢类型
- 当前计时状态
- 历史会话记录
- 统计数据

## 7. 开始计时

当某个包厢处于空闲状态时，可以点击该包厢卡片上的“开始”按钮开启新会话。

### 7.1 开始计时弹窗

点击“开始”后，系统会弹出“开始计时模式选择”窗口。

操作员需要选择两种模式之一：

- 自由计时
- 团购套餐

同时可以填写一段可选备注，例如：

- 团购码
- 顾客要求
- 赠送时长
- 前台说明

### 7.2 自由计时模式

“自由计时”是正计时模式，适用于先玩后结账场景。

开始后系统会：

- 创建一条新会话记录
- 把设备状态改为“使用中”
- 启动正计时

### 7.3 团购套餐模式

“团购套餐”是倒计时模式，适用于固定时长套餐。

可选方式包括：

- 快捷时长按钮：30 / 60 / 90 / 120 / 150 / 180 分钟
- 自定义分钟数：1 到 600 分钟

开始后系统会：

- 创建一条新会话记录
- 记录套餐总秒数
- 把设备状态改为“使用中”
- 启动倒计时

### 7.4 备注记录

开始计时时填写的备注会保存到本次会话中，并在结算时继续带入，后续也可在统计报表中修改。

## 8. 计时显示规则

### 8.1 自由计时

自由计时采用正计时方式。

显示逻辑：

- 卡片主时间显示已使用时长
- 状态提示显示“已玩 HH:MM:SS”

### 8.2 团购套餐

团购套餐采用倒计时方式。

显示逻辑：

- 卡片主时间显示剩余时长
- 状态提示显示“剩余 HH:MM:SS”
- 超时后显示负时间，例如 `-00:05:12`

### 8.3 暂停时间处理

无论是自由计时还是倒计时，暂停期间的时间不会计入实际使用时长。

系统会在恢复时自动累计暂停总时长，保证统计结果更准确。

## 9. 暂停与继续

### 9.1 暂停

当包厢处于“使用中”时，可以点击“暂停”。

暂停后系统会：

- 停止界面计时刷新
- 记录暂停开始时间
- 将会话状态改为 `paused`
- 将设备状态改为“已暂停”

### 9.2 继续

当包厢处于“已暂停”时，可以点击“继续”。

继续后系统会：

- 计算本次暂停时长
- 累加到会话暂停总时长
- 恢复计时
- 将状态切回“使用中”

## 10. 套餐加时

### 10.1 适用范围

“加时”按钮只在倒计时模式下显示，适用于以下状态：

- 使用中
- 已暂停
- 已到时

### 10.2 加时逻辑

点击“加时”后，系统会弹出续费加时窗口。

操作员可为当前套餐追加新的分钟数。加时完成后：

- 套餐总时长会增加
- 数据库中的 `countdown_seconds` 会同步更新
- 如果原来已经到时，追加时长后状态会恢复为“使用中”

### 10.3 使用场景

适合以下场景：

- 顾客续玩
- 套餐补时
- 店内赠送时长

## 11. 到时提醒与语音播报

### 11.1 即将到时提醒

对于倒计时模式，当剩余时间小于等于 5 分钟时，系统会触发预警。

预警阈值配置见：

- `COUNTDOWN_WARNING_SECS = 300`

即 300 秒，也就是 5 分钟。

### 11.2 时间到提醒

当倒计时剩余时间小于等于 0 时：

- 设备状态会变为“已到时”
- 卡片会闪烁提示
- 系统会进行语音播报

### 11.3 语音播报机制

当前版本使用 Windows 自带语音接口进行播报，无需额外安装语音库。

语音播报场景包括：

- 某包厢即将到时
- 某包厢已经到时，请及时结账

### 11.4 注意事项

语音提醒依赖 Windows 环境。如果系统本机禁用了相关语音能力，播报失败不会影响主程序继续运行。

## 12. 结算功能

### 12.1 结算入口

以下状态下可点击“结账”：

- 使用中
- 已暂停
- 已到时

### 12.2 结算前处理

弹出结算窗口前，系统会先冻结当前计时器，确保结算窗口中显示的时长与最终保存的数据一致。

如果用户取消结算：

- 正在运行中的计时会恢复
- 数据不会被写入为结束状态

### 12.3 结算窗口内容

结算窗口会显示以下信息：

- 包厢名称
- 包厢类型
- 计时模式
- 开始时间
- 实际时长
- 格式化时长
- 套餐时长（仅倒计时模式）
- 超时时长或剩余时长（仅倒计时模式）

### 12.4 付款状态

结算时可以勾选：

- `已付款`

如果不勾选，则表示本次会话已结束，但尚未付款。

### 12.5 收款方式

勾选“已付款”后，可进一步选择收款方式，支持多选：

- 美团
- 抖音
- 现金

最终会以逗号分隔方式保存，例如：

- `美团`
- `现金`
- `美团,现金`

### 12.6 备注

结算窗口可填写或补充备注，用于记录：

- 赠送时长
- 特殊情况
- 处理说明
- 顾客备注

### 12.7 结算确认后的处理

确认结算后系统会：

- 停止当前计时
- 记录结束时间
- 记录总使用秒数
- 保存付款状态
- 保存收款方式
- 保存备注
- 将会话状态改为 `completed`
- 将设备状态恢复为“空闲”

## 13. 统计报表

统计报表页面用于按日期查看历史使用情况和基础经营数据。

### 13.1 查询条件

支持以下日期方式：

- 今日
- 近 7 天
- 本月
- 自定义开始日期与结束日期

### 13.2 顶部指标

统计页会展示以下关键指标：

- 总场次
- 总时长
- 倒计时占比
- 已支付数量

说明：

- 总时长以小时显示
- 倒计时占比 = 倒计时场次 / 总场次
- 已支付数量 = 已勾选付款的已完成会话数量

### 13.3 设备维度概览

设备统计表会显示每个包厢在选定时间段内的汇总信息，包括：

- 设备名称
- 设备类型
- 场次
- 总时长
- 模式分布

模式分布示例：

- 倒计时 3 / 自由 5

### 13.4 历史记录表

历史记录表显示每次已完成会话，包含：

- 包厢
- 类型
- 模式
- 开始时间
- 时长
- 付款状态
- 收款方式
- 备注

### 13.5 备注修改

统计页支持直接修改历史记录中的“备注”列。

修改后系统会立即保存到数据库。如果保存失败，系统会提示并重新刷新数据。

### 13.6 统计页侧边摘要

统计页右侧还有两个辅助摘要：

- 支付状态
- 模式摘要

可帮助值班人员快速了解：

- 已支付数量
- 未支付数量
- 有备注的记录数量
- 倒计时与自由计时的分布
- 当前查询日期范围

## 14. 系统设置

系统设置页面用于维护基础资料，当前包含两个标签页：

1. 包厢类型管理
2. 包厢管理

设置页面现在是主窗口内嵌页面。点击左侧导航 `设` 即可进入，不会再弹出单独窗口。

### 14.1 包厢类型管理

可新增和删除包厢类型。

默认类型包括：

- PS4包厢
- PS5包厢
- Xbox包厢
- Switch包厢
- 其他包厢

### 14.2 新增包厢类型

操作步骤：

1. 打开左侧导航 `设`。
2. 进入 `包厢类型管理` 标签页。
3. 输入类型名称。
4. 点击“添加”。

新增成功后：

- 类型列表会刷新。
- 包厢管理中的类型下拉框会同步刷新。
- 控制台和统计页会在需要时刷新。

### 14.3 删除包厢类型

删除类型前系统会提示确认。

需要注意：

- 如果已有包厢引用该类型，删除可能失败或影响相关包厢显示。
- 当前版本没有更复杂的引用保护和类型合并功能。

建议：

- 已投入使用的类型不要随意删除。
- 如果只是名称不合适，建议后续通过程序升级增加编辑能力。

### 14.4 包厢管理

可新增和删除包厢设备。

每个包厢至少包含：

- 名称
- 类型

### 14.5 新增包厢

操作步骤：

1. 打开左侧导航 `设`。
2. 进入 `包厢管理` 标签页。
3. 输入包厢名称。
4. 选择包厢类型。
5. 点击“添加”。

新增成功后：

- 控制台中会新增对应设备卡片。
- 新增包厢默认排在设备列表末尾。
- 统计页中后续可统计该设备数据。

### 14.6 删除包厢

删除包厢时系统会提示确认。

删除后的影响：

- 该包厢卡片不再显示。
- 历史会话记录仍保留在数据库中。

适用于：

- 设备下线
- 设备更名后重新建档
- 门店布局调整

## 15. 程序重启后的自动恢复

本系统支持恢复未结束会话。

当程序重新启动时，会自动读取数据库中状态为以下值的会话：

- `active`
- `paused`

恢复逻辑如下：

- 如果是自由计时，会继续累计已用时长。
- 如果是倒计时，会恢复剩余时长。
- 如果恢复时发现倒计时本应到时，则会直接进入“已到时”状态。
- 如果之前是暂停状态，会恢复为“已暂停”。

这个功能可避免以下情况导致的数据丢失：

- 程序意外关闭
- 电脑重启
- 突然断电后重新启动

## 16. 日常操作建议流程

### 16.1 顾客到店开台

建议流程：

1. 在控制台找到空闲包厢。
2. 点击“开始”。
3. 选择“自由计时”或“团购套餐”。
4. 如有需要填写备注。
5. 确认开始。

### 16.2 顾客临时离开

建议流程：

1. 点击当前包厢“暂停”。
2. 顾客回来后点击“继续”。

### 16.3 顾客需要续玩

如果是套餐模式：

1. 点击“加时”。
2. 选择追加时长。
3. 确认加时。

如果是自由计时：

- 无需加时，继续保持运行即可。

### 16.4 顾客离店结算

建议流程：

1. 点击“结账”。
2. 核对开始时间和使用时长。
3. 勾选是否已付款。
4. 选择收款方式。
5. 填写备注。
6. 点击确认结束。

### 16.5 调整包厢显示顺序

建议流程：

1. 打开“设备控制台”。
2. 用鼠标按住要移动的包厢卡片。
3. 拖到目标卡片位置或列表空白位置。
4. 松开鼠标。
5. 系统会自动保存新顺序。

### 16.6 店长查看报表

建议流程：

1. 打开“统计报表”。
2. 选择“今日 / 近7天 / 本月 / 自定义”。
3. 点击“查询”。
4. 查看总体指标。
5. 查看设备使用情况。
6. 查看历史记录并补充备注。

### 16.7 维护基础资料

建议流程：

1. 打开左侧导航 `设`。
2. 在 `包厢类型管理` 中维护类型。
3. 在 `包厢管理` 中新增或删除包厢。
4. 返回 `控` 查看设备卡片是否符合预期。

主要字段：

- `id`
- `device_id`
- `timer_mode`
- `countdown_seconds`
- `start_time`
- `end_time`
- `pause_duration`
- `total_seconds`
- `paid`
- `payment_method`
- `note`
- `status`
- `created_at`

- `paid`：是否已付款。
- `payment_method`：收款方式，多个方式用逗号分隔。

### 17.4 暂停记录表 `pause_records`

主要字段：

- `id`
- `session_id`
- `pause_start`
- `pause_end`

## 20. 使用注意事项

### 20.1 删除操作需谨慎

删除包厢类型和包厢前都应确认是否还在实际使用中。

### 20.2 本系统以“时长记录”为主

若门店需要自动收银金额、价格套餐、优惠折扣或找零计算，需要在后续版本中增加价格和金额计算模块。

### 20.3 语音提醒仅在 Windows 下工作最佳

由于当前使用的是 Windows 自带语音接口，其他平台兼容性有限。

### 20.4 数据库建议定期备份

建议定期备份以下文件：

- `data/videogame.db`

尤其在以下场景前建议先备份：

- 升级程序
- 批量调整设备
- 重装系统
- 迁移到其他电脑

### 20.5 不要直接编辑数据库

除非明确知道字段含义，否则不建议直接修改 SQLite 数据库文件。手动改动可能造成：

- 设备状态与会话状态不一致
- 计时恢复异常
- 历史统计不准确
- 包厢排序异常

## 21. 结语
